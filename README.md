# Her Desk - Java 远程桌面工具

纯 Java 实现的远程桌面工具，**一个 JAR** 同时包含被控端、控制端与公网中继，适用于 Windows / macOS，JDK 8 ~ JDK 21 均可运行。

支持 **内网直连** 与 **公网中继** 两种连接方式，无需浏览器、无需安装客户端以外的依赖。

## 功能特性

### 连接与网络

- **内网直连**：TCP 直连，同一局域网内 IP 互通即可，默认端口 **5900**
- **公网中继**：通过 VPS 中转，突破局域网限制，控制端与被控端均主动连中继
- **房间密码**：中继模式需填写房间密码（当前默认 `123456`），密码错误拒绝接入
- **随机房间号**：被控端中继模式自动生成 8 位随机房间号，可点击「重新生成」
- **连接超时**：网络连接等待 **15 秒**，超时后日志面板给出详细原因提示

### 画面与性能

- 增量差分画面传输（96×96 区块），首帧全屏，后续局部刷新
- JPEG 压缩，三档画质：高清晰 / 均衡 / 流畅
- 静止画面自动降频，操作后自动提速（最高约 50 FPS）
- macOS Retina 自动缩放，坐标自动换算

### 控制与交互

- 完整键鼠控制（移动、点击、滚轮、组合键）
- 控制端 **全屏模式**：独占全屏显示远程画面，右上角「退出全屏」按钮，**Esc** 也可退出
- 单客户端独占连接

### 界面与日志

- 统一浅蓝主题 UI（启动器 / 被控端 / 控制端）
- 被控端、控制端均内置 **日志面板**，记录连接步骤与网络错误
- 控制端底部显示实时 **帧率（FPS）**

### 部署

- 一个 JAR 双击即用，Swing 轻量界面
- 独立中继 JAR（`her-desk-relay.jar`），可部署在公网 VPS
- 支持 `jpackage` 打包为 **macOS DMG** / **Windows EXE**（内置 JRE，用户无需安装 Java）

## 环境要求

| 项目 | 要求 |
|------|------|
| JDK | 8 ~ 21（推荐 11 或 17） |
| 操作系统 | Windows 10/11 或 macOS 10.15+ |
| 内网直连 | 同一局域网，被控端与控制端 IP 互通 |
| 公网中继 | VPS 开放中继 TCP 端口（如 9000、11111） |
| 默认端口 | 内网直连 **5900**（TCP，可在被控端界面修改） |

### 角色说明

| 角色 | 说明 | 典型场景 |
|------|------|----------|
| **被控端（Server）** | 被远程控制的电脑，负责截图 + 执行键鼠 | 办公室电脑、家里 Mac |
| **控制端（Client）** | 发起连接、显示画面、发送键鼠操作 | 你的笔记本 |
| **中继（Relay）** | 公网 VPS 上的配对服务，转发两端流量 | 跨网访问时部署 |

> **权限重点**：截图和模拟键鼠发生在 **被控端**，因此 **macOS 的屏幕录制 / 辅助功能权限主要在被控端配置**。控制端一般无需特殊权限。

## 快速开始

### 内网直连（同一局域网）

```
1. 两台电脑连同一内网
2. 被控端：构建 JAR → 配置权限 → 启动「被控端」→ 点「启动服务」
3. 控制端：启动「控制端」→ 连接模式选「内网直连」→ 输入被控端 IP + 端口 5900 → 点「连接」
4. 在画面区域用鼠标、键盘远程操作
```

### 公网中继（跨网访问）

```
1. VPS：java -jar her-desk-relay.jar 9000
2. 被控端：连接模式选「公网中继」→ 填中继地址/端口 → 记下自动生成的房间号 → 密码填 123456 → 启动服务
3. 控制端：连接模式选「公网中继」→ 填相同中继地址/端口/房间号/密码 → 连接
4. 可选：点「全屏」进入独占全屏模式
```

## 构建与运行

### 方式一：Maven 完整打包（推荐）

```bash
mvn clean package
```

产物：

```
target/her-desk-1.0.0.jar      # 被控端 + 控制端（主程序）
target/her-desk-relay.jar      # 公网中继（部署在 VPS）
```

### 方式二：仅编译 + 手动打 JAR（Nexus 403 等打包失败时）

若 `mvn package` 因私服权限返回 403，可改用：

```bash
mvn compile
```

然后手动打包：

```bash
# 主程序 JAR
cd target/classes
jar cfm ../her-desk-1.0.0.jar ../MANIFEST.MF .

# 中继 JAR（需先有 target/RELAY-MANIFEST.MF）
jar cfm ../her-desk-relay.jar ../RELAY-MANIFEST.MF com/herdesk/relay/ com/herdesk/common/
```

`MANIFEST.MF` 与 `RELAY-MANIFEST.MF` 位于 `target/` 目录（首次 `mvn compile` 后可从仓库历史或本地生成）。

### 方式三：双击启动脚本

| 脚本 | 平台 | 作用 |
|------|------|------|
| `scripts/run.command` | macOS | 模式选择 |
| `scripts/run-server.command` | macOS | 被控端 |
| `scripts/run-client.command` | macOS | 控制端 |
| `scripts/run.bat` | Windows | 模式选择 |
| `scripts/run-server.bat` | Windows | 被控端 |
| `scripts/run-client.bat` | Windows | 控制端 |
| `scripts/run-relay.command` | macOS / Linux | 公网中继服务 |
| `scripts/run-relay.bat` | Windows | 公网中继服务 |

脚本会自动查找 `target/her-desk-*.jar`，未找到时会提示先执行构建。

**macOS 首次运行 `.command` 脚本**若提示无法打开：

```bash
chmod +x scripts/*.command
```

或在 **系统设置 → 隐私与安全性** 中允许该脚本运行。

### 方式四：命令行运行 JAR

```bash
java -jar target/her-desk-1.0.0.jar          # 弹出模式选择
java -jar target/her-desk-1.0.0.jar server   # 被控端
java -jar target/her-desk-1.0.0.jar client   # 控制端
java -jar target/her-desk-relay.jar 9000     # 公网中继（VPS，端口可改）
```

### 方式五：IDE 中运行

在 IntelliJ IDEA / Cursor 中直接运行 `com.herdesk.Launcher` 主类，或通过 Maven 插件启动。  
**注意**：IDE 运行时，系统权限里要勾选的是 **IDE 本身**（见下文 macOS 权限章节）。

### 方式六：原生安装包打包（DMG / EXE）

面向最终用户分发时，可使用 JDK 自带的 `jpackage` 打包为**自带运行时的原生安装包**，用户**无需单独安装 Java**。

#### 环境要求

| 项目 | 要求 |
|------|------|
| 打包工具 | **JDK 17+**（须包含 `jpackage` 命令） |
| macOS 产物 | 须在 **macOS** 上执行打包脚本 |
| Windows 产物 | 须在 **Windows** 上执行打包脚本 |
| 应用 JAR | 脚本会自动 `mvn compile` 并手动打 JAR |

> **注意**：`jpackage` 不能跨平台交叉打包。在 Mac 上只能生成 `.dmg`，在 Windows 上只能生成 `.exe`。

#### macOS：生成 DMG

```bash
chmod +x scripts/package-mac.command
./scripts/package-mac.command
```

产物目录 `target/dist/`：

```
Her Desk-1.0.0.dmg          # 主程序（被控端 + 控制端）
Her Desk Relay-1.0.0.dmg    # 公网中继服务
```

安装：双击 DMG → 将应用拖入「应用程序」文件夹 → 从启动台打开。

**首次打开提示「无法验证开发者」**：右键应用 → **打开**（当前安装包未做苹果代码签名，属正常现象）。

#### Windows：生成 EXE

在 Windows 电脑上执行：

```bat
scripts\package-windows.bat
```

产物目录 `target\dist\`：

```
Her Desk-1.0.0.exe          # 主程序安装包
Her Desk Relay-1.0.0.exe    # 中继服务安装包
```

按安装向导完成安装后，从开始菜单或桌面快捷方式启动。

#### 打包脚本说明

| 脚本 | 平台 | 作用 |
|------|------|------|
| `scripts/package-mac.command` | macOS | 一键编译 JAR 并生成两个 DMG |
| `scripts/package-windows.bat` | Windows | 一键编译 JAR 并生成两个 EXE |

脚本流程：编译 → 打 JAR → 调用 `jpackage` → 输出到 `target/dist/`。

#### 与 JAR 方式对比

| 对比项 | JAR 方式 | DMG / EXE 安装包 |
|--------|----------|------------------|
| 用户是否需装 Java | 需要 JDK 8+ | **不需要**（内置 JRE） |
| 体积 | 约 120 KB | 约 50 MB / 个 |
| 适用场景 | 开发、技术用户 | **商业分发、普通用户** |
| 启动方式 | `java -jar` 或脚本 | 双击应用图标 |

#### 常见问题

**Q: `jpackage` 命令找不到？**  
A: 请安装 JDK 17 或更高版本，并确保 `JAVA_HOME/bin` 在 PATH 中。macOS 可用 SDKMAN：`sdk install java 17.0.8-tem`。

**Q: 在 Mac 上能否直接打 Windows 的 EXE？**  
A: 不能。请在 Windows 机器上运行 `scripts\package-windows.bat`，或使用 Windows CI 构建。

**Q: DMG/EXE 需要提交到 Git 吗？**  
A: 不需要。`target/dist/` 已在 `.gitignore` 中忽略，仅提交打包脚本即可。

## 界面说明

### 启动器

启动 `her-desk-1.0.0.jar` 后选择：

- **被控端（Server）**：本机作为被远程控制的电脑
- **控制端（Client）**：本机作为远程操作端

### 连接模式

两端界面均有 **「连接模式」** 下拉框：

| 模式 | 适用场景 |
|------|----------|
| **内网直连** | 同一局域网，填被控端 IP + 端口 |
| **公网中继** | 跨网访问，填中继地址 + 端口 + 房间号 + 密码 |

### 被控端界面字段

| 模式 | 字段 |
|------|------|
| 内网直连 | 监听端口（默认 5900） |
| 公网中继 | 中继地址、中继端口、房间号（自动生成）、房间密码 |

按钮：**启动服务** / **停止服务**；底部有状态栏、FPS、日志面板。

### 控制端界面字段

| 模式 | 字段 |
|------|------|
| 内网直连 | 被控端 IP、端口 |
| 公网中继 | 中继地址、中继端口、房间号、房间密码 |

按钮：**连接** / **断开** / **全屏**；可切换画质档位；底部有状态栏、FPS、日志面板。

### 全屏模式

连接成功后，控制端可点击 **「全屏」**：

- 进入独占全屏，远程画面铺满屏幕
- 右上角显示 **「退出全屏」** 按钮
- 按 **Esc** 也可退出全屏
- 全屏下键鼠操作仍发送至被控端

### 日志面板

被控端、控制端底部均有日志区域，记录：

- 用户操作（启动服务、发起连接等）
- 中继握手步骤（REGISTER / JOIN）
- 网络错误详情（超时、拒绝连接、房间未找到、密码错误等）

## 公网中继跨网访问

通过公网服务器中转，可突破局域网限制（HDRD 协议透明转发，无需修改画面/键鼠逻辑）。

### 架构

```
控制端 ──主动连接──► 公网 Relay ◄──主动连接── 被控端
              （相同房间号 + 密码配对后双向转发）
```

> **重要**：公网中继模式下，**控制端和被控端都要主动连到 Relay 服务端**，不是只连一端，也不是被控端暴露公网端口。

### 在哪里配置中继地址（界面操作）

**无需改代码或配置文件**，在 Her Desk 图形界面里填写即可。

启动程序后，在窗口中找到 **「连接模式」** 下拉框，选择 **「公网中继」**，会出现四个输入项：

| 界面字段 | 填什么 | 示例 |
|----------|--------|------|
| **中继地址** | VPS 的公网 IP 或域名 | `123.45.67.89`、`relay.example.com` |
| **中继端口** | Relay 服务监听端口 | `9000`、`11111` |
| **房间号** | 两端必须完全一致 | 被控端自动生成，如 `a3k9m2x1` |
| **房间密码** | 两端必须完全一致 | 默认 `123456` |

界面可能预填示例中继地址，**使用前请改成你自己的 VPS 地址**。

#### 被控端在哪里填？

1. 启动 JAR → 选择 **「被控端（Server）」**
2. **连接模式** → 选 **「公网中继」**
3. 填写 **中继地址**、**中继端口**；房间号会自动生成（可点「重新生成」）
4. 填写 **房间密码**（默认 `123456`）
5. 点击 **「启动服务」**
6. 日志显示「中继注册成功」表示已连上 Relay

#### 控制端在哪里填？

1. 启动 JAR → 选择 **「控制端（Client）」**
2. **连接模式** → 选 **「公网中继」**
3. 填写 **相同** 的 **中继地址**、**中继端口**、**房间号**、**房间密码**
4. 点击 **「连接」**

#### 配置注意事项

| 要点 | 说明 |
|------|------|
| 两端都要填 | 控制端、被控端都必须配置并连到 Relay |
| 房间号必须一致 | 被控端生成后告知控制端，或双方约定固定房间号 |
| 密码必须一致 | 默认 `123456`，填错会报「房间密码错误」 |
| 中继地址填 VPS 公网地址 | **不要**填被控端本机 `192.168.x.x` |
| 被控端先启动 | 须先注册房间，控制端再加入 |
| 内网直连不用 Relay | 选「内网直连」时，控制端填被控端局域网 IP 即可 |

#### 两种模式对照

| 连接模式 | 被控端配置 | 控制端配置 |
|----------|------------|------------|
| **内网直连** | 监听端口（默认 5900） | 被控端内网 IP + 端口 |
| **公网中继** | 中继地址 + 端口 + 房间号 + 密码 | 中继地址 + 端口 + 房间号 + 密码（相同） |

### 房间号与密码

| 项目 | 说明 |
|------|------|
| **房间号** | 被控端自动生成 8 位小写字母+数字；可手动修改或点「重新生成」 |
| **房间号长度** | 4 ~ 32 字符 |
| **默认密码** | `123456`（代码常量 `RelayAuth.DEFAULT_PASSWORD`） |
| **密码长度** | 1 ~ 64 字符 |
| **安全级别** | 明文传输、无 TLS；公网使用请尽快改为自定义复杂密码 |

### 1. 在公网 VPS 部署 Relay

```bash
mvn clean package
java -jar target/her-desk-relay.jar 9000
```

端口可自定义，例如：

```bash
java -jar target/her-desk-relay.jar 11111
```

**VPS 防火墙 / 安全组**：放行对应 TCP 端口入站。

Relay 控制台会输出结构化日志（注册、加入、配对、错误等）。

### 2. 被控端（公网中继模式）

1. 启动 Her Desk → **被控端**
2. 连接模式选 **「公网中继」**
3. 填写 VPS 公网地址、端口、确认房间号与密码
4. 点击 **启动服务**，日志显示注册成功

### 3. 控制端（公网中继模式）

1. 启动 Her Desk → **控制端**
2. 连接模式选 **「公网中继」**
3. 填写 **相同** 的中继地址、端口、房间号、密码
4. 点击 **连接**

### 中继控制协议

配对前使用中继专用 **文本行协议**（UTF-8，以换行结束）：

| 发送方 | 命令格式 | 成功响应 |
|--------|----------|----------|
| 被控端 | `REGISTER <房间号> <密码>` | `OK WAITING` |
| 控制端 | `JOIN <房间号> <密码>` | `OK CONNECTED` |

配对成功后，Relay 透明转发二进制 **HDRD** 流量。

#### 常见错误响应

| 响应 | 含义 | 处理 |
|------|------|------|
| `ERROR ROOM_NOT_FOUND` | 房间未注册或房间号不一致 | 确认被控端已启动、房间号一致 |
| `ERROR INVALID_PASSWORD` | 密码错误 | 确认两端密码一致 |
| 连接超时（15 秒） | 端口未开放、防火墙拦截或 Relay 未启动 | 检查 VPS 端口与安全组 |

### 安全说明

- 当前版本密码为 **明文传输**，中继与控制流量 **无 TLS 加密**
- 知道房间号 + 密码即可接入；公网使用请使用复杂房间号并修改默认密码
- 后续可扩展：可配置密码、TLS、令牌鉴权

## 权限配置（必读）

### macOS 被控端权限

被控端 Mac **必须**配置以下两项，否则会出现：

- 只能看到壁纸，看不到桌面图标、窗口、Dock
- 画面偏色
- 鼠标能点但画面不更新或更新极慢

#### 1. 屏幕录制（必须）

用于截取远程画面。

**操作路径（macOS Ventura / Sonoma / Sequoia）：**

1. 打开 **系统设置**
2. 进入 **隐私与安全性**
3. 点击 **屏幕录制**
4. 点击左下角 **🔒** 解锁（如需输入密码）
5. 勾选运行 Java 的对应程序：

| 你怎么启动被控端 | 需要勾选 |
|------------------|----------|
| 终端执行 `java -jar ...` | **终端 (Terminal)** |
| 双击 `run-server.command` | **终端 (Terminal)** |
| IntelliJ IDEA 运行 | **IntelliJ IDEA** |
| Cursor 运行 | **Cursor** |
| 列表中有 **java** 项 | 也可勾选 **java** |

6. **完全退出** Her Desk 被控端（不是只断连）
7. **重新启动**被控端
8. 控制端 **断开重连**

> macOS 修改权限后 **必须重启应用** 才生效，仅重连无效。

**如何确认权限已生效：**

- 远程画面能看到 **Dock（底部程序栏）**
- 能看到 **桌面图标、文件夹、打开的窗口**
- 点击后窗口变化能及时显示

#### 2. 辅助功能（必须）

用于在被控端模拟鼠标、键盘操作。

**操作路径：**

1. **系统设置 → 隐私与安全性 → 辅助功能**
2. 解锁并勾选与上表相同的程序（Terminal / IDE / java）
3. 退出并重启被控端

**如何确认权限已生效：**

- 控制端点击后，被控端鼠标光标会移动
- 能正常点击、输入文字

#### 3. macOS 权限排查清单

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 只有壁纸，无 Dock/图标 | 未授权屏幕录制 | 按上文勾选并重启 |
| 能看画面但点不动 | 未授权辅助功能 | 勾选辅助功能并重启 |
| 勾选了仍无效 | 勾错了程序 | 确认启动方式，勾选 Terminal 或 IDE |
| 勾选了仍无效 | 未重启应用 | 完全退出后重新启动 |
| 弹出权限提示时点拒绝 | 权限被拒绝 | 去系统设置手动打开 |

#### 4. macOS 控制端

控制端 Mac **通常不需要**屏幕录制和辅助功能。  
控制端只负责显示画面和采集本地键鼠发送给被控端。

### Windows 被控端权限

Windows **没有** macOS 那样的「屏幕录制」弹窗权限，但需要保证 **防火墙放行端口**。

#### 1. 防火墙放行（必须）

被控端需允许其他电脑通过 TCP 连接默认端口 **5900**。

**方法一：首次弹窗时允许（推荐）**

1. 被控端启动服务后，Windows 可能弹出防火墙提示
2. 勾选 **专用网络**
3. 点击 **允许访问**

**方法二：手动添加入站规则**

1. 打开 **控制面板 → Windows Defender 防火墙 → 高级设置**
2. **入站规则 → 新建规则**
3. 选择 **端口 → TCP → 特定本地端口：5900**
4. 选择 **允许连接**
5. 勾选 **专用**（内网环境）
6. 命名如 `Her Desk 5900`

**方法三：放行 Java 程序**

1. **Windows 安全中心 → 防火墙和网络保护 → 允许应用通过防火墙**
2. 找到 **Java(TM) Platform SE binary** 或 **OpenJDK**
3. 勾选 **专用网络**

#### 2. 其他 Windows 注意项

| 项目 | 说明 |
|------|------|
| 管理员权限 | 一般 **不需要** 管理员身份运行 |
| 杀毒软件 | 若连接失败，检查是否拦截了 Java 或 5900 端口 |
| 网络类型 | 确保当前网络为 **专用网络**，不是「公用网络」 |
| UAC | 不影响远程桌面基本功能 |

#### 3. Windows 控制端

控制端 Windows **无需特殊权限**，只要能 ping 通被控端 IP 即可。

## 使用步骤

根据场景选择 **内网直连** 或 **公网中继**（配置方式见 [公网中继跨网访问](#公网中继跨网访问)）。

### 场景 A：内网直连（同一局域网）

#### 第一步：在被控端电脑

1. 安装 JDK 8+，构建或获取 `her-desk-1.0.0.jar`
2. **按上文完成权限配置**（Mac 必做，Windows 放行防火墙）
3. 启动被控端（`run-server.command` / `run-server.bat`）
4. **连接模式** 保持 **「内网直连」**
5. 确认本机内网 IP，端口默认 **5900**
6. 点击 **「启动服务」**

#### 第二步：在控制端电脑

1. 启动控制端（`run-client.command` / `run-client.bat`）
2. **连接模式** 选 **「内网直连」**
3. 输入被控端 **内网 IP** 和 **端口**
4. 点击 **「连接」**，在画面区域操作键鼠
5. 可选：点击 **「全屏」** 进入独占全屏

---

### 场景 B：公网中继（跨网访问）

#### 第零步：在 VPS 启动 Relay

```bash
java -jar her-desk-relay.jar 9000
```

#### 第一步：在被控端电脑

1. 完成权限配置（Mac 屏幕录制 + 辅助功能）
2. 启动被控端
3. **连接模式** 选 **「公网中继」**
4. 填写 VPS **中继地址**、**端口**；确认 **房间号** 与 **密码**
5. 点击 **「启动服务」**，等待日志显示注册成功
6. 将房间号告知控制端操作者

#### 第二步：在控制端电脑

1. 启动控制端
2. **连接模式** 选 **「公网中继」**
3. 填写 **相同** 的中继地址、端口、房间号、密码
4. 点击 **「连接」**

---

### 断开连接

- 控制端点击 **「断开」**
- 或被控端点击 **「停止服务」**
- 全屏模式下可先按 **Esc** 或点「退出全屏」再断开

### 查看本机 IP 参考（仅内网直连需要）

| 平台 | 命令 |
|------|------|
| macOS / Linux | `ifconfig` 或 `ip addr` |
| Windows | `ipconfig` |

选择 `192.168.x.x` 或 `10.x.x.x` 开头的地址，不要使用 `127.0.0.1`。

## 协议说明

### HDRD 二进制协议（配对后）

- **传输**：TCP 长连接，内网直连默认端口 5900
- **帧格式**：`魔数 HDRD(4) + 类型(1) + 长度(4) + 载荷(N)`，大端序
- **握手响应**：`originX, originY, logicalW, logicalH, pixelW, pixelH`（24 字节）
- **消息类型**：握手、全帧、差分帧、鼠标、键盘、画质、心跳、断开

### 中继文本协议（配对前）

见 [中继控制协议](#中继控制协议)。

## 常见问题

### 连接类

**Q: 控制端和被控端在哪里改 Relay 地址？**  
A: 在各自界面 **连接模式 → 公网中继** 下填写「中继地址」「中继端口」「房间号」「房间密码」，无需改代码。

**Q: 两端都要连 Relay 吗？**  
A: 是。公网中继模式下控制端、被控端都需主动连接 VPS 上的 Relay，并用相同房间号与密码配对。

**Q: 房间号从哪里来？**  
A: 被控端选公网中继后会 **自动生成** 8 位随机房间号，可点「重新生成」。控制端须手动填入被控端显示的同一房间号。

**Q: 默认密码是什么？**  
A: `123456`。两端必须一致，否则会提示「房间密码错误」。

**Q: 如何从外网连接家里/公司电脑？**  
A: 在公网 VPS 运行 `her-desk-relay.jar`，被控端和控制端均选「公网中继」，填写相同房间号与密码。详见 [公网中继跨网访问](#公网中继跨网访问)。

**Q: 提示 ROOM_NOT_FOUND？**  
A: 被控端未启动或未成功注册到中继；确认被控端先启动、房间号一致、VPS 中继端口可达。

**Q: 提示房间密码错误？**  
A: 控制端与被控端填写的密码不一致；默认应为 `123456`。

**Q: 连接失败 / 超时？**

1. 确认被控端已点「启动服务」
2. 确认 IP、端口正确（内网默认 5900）
3. 公网中继：确认 VPS 端口已放行、Relay 进程在运行
4. 两台电脑是否同一内网（内网模式可 `ping` 被控端 IP）
5. Windows 被控端检查防火墙是否放行 5900
6. 查看日志面板中的详细错误说明（15 秒超时）

**Q: 提示「拒绝新连接：已有客户端占用」？**

同时只允许一个控制端连接。先断开已有连接，或重启被控端服务。

---

### 画面类

**Q: 只能看到壁纸，看不到 Dock、桌面图标、窗口？（Mac）**

几乎一定是 **屏幕录制权限未授权或未生效**。  
请按 [macOS 被控端权限](#macos-被控端权限) 配置，**重启被控端** 后重连。

**Q: 全屏后黑屏、看不到退出按钮？**

请使用最新版本 JAR。当前版全屏使用 `BorderLayout` + `GlassPane` 实现，应能正常显示画面与右上角「退出全屏」；也可按 **Esc** 退出。

**Q: 画面偏粉 / 偏紫？（Mac）**

1. 确认屏幕录制权限已授予
2. 重新构建并运行最新版 JAR

**Q: 画面卡顿？**

1. 画质切为 **「流畅」**
2. 确认网络带宽充足（公网中继注意 VPS 上行）
3. 被控端避免同时运行高负载程序

**Q: 点击后有反应，但画面很久才更新？**

1. Mac 检查屏幕录制权限
2. 重新连接触发全帧刷新
3. 画质切为「流畅」

---

### 键鼠类

**Q: 能看画面但鼠标/键盘控制无效？（Mac）**

检查 **辅助功能** 权限，勾选正确程序后重启被控端。

**Q: 鼠标位置有偏移？**

1. 确认使用最新版本
2. Mac Retina / 多屏场景可能有 1~2 像素偏差
3. 建议单屏测试

**Q: Mac 上 Command 键行为不对？**

工具会自动做 Command ↔ Control 的简单映射。跨系统（Mac 控 Windows 或反向）部分快捷键可能不完全一致。

```bash
chmod +x scripts/*.command
```

```bash
java -jar target/her-desk-1.0.0.jar server
```

**Q: mvn package 报 Nexus 403？**

改用 `mvn compile` + 手动 `jar cfm` 打包，见 [方式二](#方式二仅编译--手动打-jarnexus-403-等打包失败时)。

**Q: 找不到 java 命令？**

安装 JDK 并确认：

```bash
java -version
```
