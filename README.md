# Her Desk - 内网 Java 远程桌面工具

纯 Java 实现的内网远程桌面工具，**一个 JAR** 同时包含被控端与控制端，适用于 Windows / macOS，JDK 8 ~ JDK 21 均可运行。

## 功能特性

- TCP 直连，无需公网、无需中转服务器
- 增量差分画面传输（96×96 区块），首帧全屏，后续局部刷新
- JPEG 压缩，三档画质：高清晰 / 均衡 / 流畅
- 静止画面自动降频，操作后自动提速（最高约 50fps）
- 完整键鼠控制（移动、点击、滚轮、组合键）
- macOS Retina 自动缩放，坐标自动换算
- 单客户端独占连接
- 一个 JAR 双击即用，Swing 轻量界面

## 环境要求

| 项目 | 要求 |
|------|------|
| JDK | 8 ~ 21（推荐 11 或 17） |
| 操作系统 | Windows 10/11 或 macOS 10.15+ |
| 网络 | 同一局域网，被控端与控制端 IP 互通 |
| 默认端口 | **5900**（TCP，可在被控端界面修改） |

### 角色说明

| 角色 | 说明 | 典型场景 |
|------|------|----------|
| **被控端（Server）** | 被远程控制的电脑，负责截图 + 执行键鼠 | 办公室电脑、家里 Mac |
| **控制端（Client）** | 发起连接、显示画面、发送键鼠操作 | 你的笔记本 |

> **权限重点**：截图和模拟键鼠发生在 **被控端**，因此 **macOS 的屏幕录制 / 辅助功能权限主要在被控端配置**。控制端一般无需特殊权限。

## 快速开始

```
1. 两台电脑连同一内网
2. 被控端：构建 JAR → 配置权限 → 启动「被控端」→ 点「启动服务」
3. 控制端：启动「控制端」→ 输入被控端 IP + 端口 5900 → 点「连接」
4. 在画面区域用鼠标、键盘远程操作
```

## 构建与运行

### 构建

```bash
mvn clean package
```

产物：

```
target/her-desk-1.0.0.jar
```

### 方式一：双击启动脚本（推荐）

| 脚本 | 平台 | 作用 |
|------|------|------|
| `scripts/run.command` | macOS | 模式选择 |
| `scripts/run-server.command` | macOS | 被控端 |
| `scripts/run-client.command` | macOS | 控制端 |
| `scripts/run.bat` | Windows | 模式选择 |
| `scripts/run-server.bat` | Windows | 被控端 |
| `scripts/run-client.bat` | Windows | 控制端 |

脚本会自动查找 `target/her-desk-*.jar`，未找到时会提示先执行 `mvn package`。

**macOS 首次运行 `.command` 脚本**若提示无法打开：

```bash
chmod +x scripts/*.command
```

或在 **系统设置 → 隐私与安全性** 中允许该脚本运行。

### 方式二：命令行运行 JAR

```bash
java -jar target/her-desk-1.0.0.jar          # 弹出模式选择
java -jar target/her-desk-1.0.0.jar server   # 被控端
java -jar target/her-desk-1.0.0.jar client   # 控制端
```

### 方式三：IDE 中运行

在 IntelliJ IDEA / Cursor 中直接运行 `com.herdesk.Launcher` 主类，或通过 Maven 插件启动。  
**注意**：IDE 运行时，系统权限里要勾选的是 **IDE 本身**（见下文 macOS 权限章节）。

## 权限配置（必读）

### macOS 被控端权限

被控端 Mac **必须**配置以下两项，否则会出现：

- 只能看到壁纸，看不到桌面图标、窗口、Dock
- 画面偏色
- 鼠标能点但画面不更新或更新极慢

#### 1. 屏幕录制（必须）

用于截取远程画面。

**操作路径（macOS Ventura / Sonoma / Sequoia）：**

1. 打开 **系统设置**
2. 进入 **隐私与安全性**
3. 点击 **屏幕录制**
4. 点击左下角 **🔒** 解锁（如需输入密码）
5. 勾选运行 Java 的对应程序：

| 你怎么启动被控端 | 需要勾选 |
|------------------|----------|
| 终端执行 `java -jar ...` | **终端 (Terminal)** |
| 双击 `run-server.command` | **终端 (Terminal)** |
| IntelliJ IDEA 运行 | **IntelliJ IDEA** |
| Cursor 运行 | **Cursor** |
| 列表中有 **java** 项 | 也可勾选 **java** |

6. **完全退出** Her Desk 被控端（不是只断连）
7. **重新启动**被控端
8. 控制端 **断开重连**

> macOS 修改权限后 **必须重启应用** 才生效，仅重连无效。

**如何确认权限已生效：**

- 远程画面能看到 **Dock（底部程序栏）**
- 能看到 **桌面图标、文件夹、打开的窗口**
- 点击后窗口变化能及时显示

#### 2. 辅助功能（必须）

用于在被控端模拟鼠标、键盘操作。

**操作路径：**

1. **系统设置 → 隐私与安全性 → 辅助功能**
2. 解锁并勾选与上表相同的程序（Terminal / IDE / java）
3. 退出并重启被控端

**如何确认权限已生效：**

- 控制端点击后，被控端鼠标光标会移动
- 能正常点击、输入文字

#### 3. macOS 权限排查清单

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 只有壁纸，无 Dock/图标 | 未授权屏幕录制 | 按上文勾选并重启 |
| 能看画面但点不动 | 未授权辅助功能 | 勾选辅助功能并重启 |
| 勾选了仍无效 | 勾错了程序 | 确认启动方式，勾选 Terminal 或 IDE |
| 勾选了仍无效 | 未重启应用 | 完全退出后重新启动 |
| 弹出权限提示时点拒绝 | 权限被拒绝 | 去系统设置手动打开 |

#### 4. macOS 控制端

控制端 Mac **通常不需要**屏幕录制和辅助功能。  
控制端只负责显示画面和采集本地键鼠发送给被控端。

---

### Windows 被控端权限

Windows **没有** macOS 那样的「屏幕录制」弹窗权限，但需要保证 **防火墙放行端口**。

#### 1. 防火墙放行（必须）

被控端需允许其他电脑通过 TCP 连接默认端口 **5900**。

**方法一：首次弹窗时允许（推荐）**

1. 被控端启动服务后，Windows 可能弹出防火墙提示
2. 勾选 **专用网络**
3. 点击 **允许访问**

**方法二：手动添加入站规则**

1. 打开 **控制面板 → Windows Defender 防火墙 → 高级设置**
2. **入站规则 → 新建规则**
3. 选择 **端口 → TCP → 特定本地端口：5900**
4. 选择 **允许连接**
5. 勾选 **专用**（内网环境）
6. 命名如 `Her Desk 5900`

**方法三：放行 Java 程序**

1. **Windows 安全中心 → 防火墙和网络保护 → 允许应用通过防火墙**
2. 找到 **Java(TM) Platform SE binary** 或 **OpenJDK**
3. 勾选 **专用网络**

#### 2. 其他 Windows 注意项

| 项目 | 说明 |
|------|------|
| 管理员权限 | 一般 **不需要** 管理员身份运行 |
| 杀毒软件 | 若连接失败，检查是否拦截了 Java 或 5900 端口 |
| 网络类型 | 确保当前网络为 **专用网络**，不是「公用网络」 |
| UAC | 不影响远程桌面基本功能 |

#### 3. Windows 控制端

控制端 Windows **无需特殊权限**，只要能 ping 通被控端 IP 即可。

## 使用步骤

### 第一步：在被控端电脑

1. 安装 JDK 8+，构建或获取 `her-desk-1.0.0.jar`
2. **按上文完成权限配置**（Mac 必做，Windows 放行防火墙）
3. 启动被控端：
   - Mac：双击 `scripts/run-server.command`
   - Windows：双击 `scripts/run-server.bat`
4. 确认界面显示的 **本机内网 IP**（如 `192.168.1.105`）
5. 端口保持默认 **5900**（或自定义）
6. 点击 **「启动服务」**，状态显示「等待连接...」

### 第二步：在控制端电脑

1. 启动控制端：
   - Mac：双击 `scripts/run-client.command`
   - Windows：双击 `scripts/run-client.bat`
2. 输入被控端 **内网 IP** 和 **端口**
3. 点击 **「连接」**
4. 连接成功后，**用鼠标点击画面区域** 获取焦点
5. 即可远程操作；顶部可切换画质

### 第三步：断开连接

- 控制端点击 **「断开」**
- 或被控端点击 **「停止服务」**

### 查看本机 IP 参考

| 平台 | 命令 |
|------|------|
| macOS / Linux | `ifconfig` 或 `ip addr` |
| Windows | `ipconfig` |

选择 `192.168.x.x` 或 `10.x.x.x` 开头的地址，不要使用 `127.0.0.1`。

## 画质与流畅度

| 档位 | JPEG 质量 | 适用场景 |
|------|-----------|----------|
| 高清晰 | 85% | 看文字、写代码 |
| 均衡 | 70% | 默认，日常使用 |
| 流畅 | 50% | 网络一般或感觉卡顿时 |

**卡顿优化建议：**

1. 切换为 **「流畅」** 画质
2. 关闭被控端不必要的动画/透明效果
3. 内网尽量使用有线或 5GHz Wi-Fi
4. 控制端底部状态栏可查看实时 **帧率（FPS）**

## 多显示器与 Retina

| 场景 | 行为 |
|------|------|
| macOS 单屏 Retina | 自动缩放到逻辑分辨率，坐标自动换算 |
| macOS 多屏 | 当前版本捕获 **主屏**，副屏内容不在画面中 |
| Windows 多屏 | 捕获 **虚拟桌面**（所有显示器合并区域） |

若鼠标位置有轻微偏移，多为混合 DPI 或多屏布局导致，建议先用单屏测试。

## 协议说明

- **传输**：TCP 长连接，默认端口 5900
- **帧格式**：`魔数 HDRD(4) + 类型(1) + 长度(4) + 载荷(N)`，大端序
- **握手响应**：`originX, originY, logicalW, logicalH, pixelW, pixelH`（24 字节）
- **消息类型**：握手、全帧、差分帧、鼠标、键盘、画质、心跳、断开

## 常见问题

### 连接类

**Q: 连接失败 / 超时？**

1. 确认被控端已点「启动服务」
2. 确认 IP、端口正确（默认 5900）
3. 两台电脑是否同一内网（可 `ping` 被控端 IP）
4. Windows 被控端检查防火墙是否放行 5900
5. 公司网络是否隔离了设备互访

**Q: 提示「拒绝新连接：已有客户端占用」？**

同时只允许一个控制端连接。先断开已有连接，或重启被控端服务。

---

### 画面类

**Q: 只能看到壁纸，看不到 Dock、桌面图标、窗口？（Mac）**

几乎一定是 **屏幕录制权限未授权或未生效**。  
请按 [macOS 被控端权限](#macos-被控端权限) 配置，**重启被控端** 后重连。

**Q: 画面偏粉 / 偏紫？（Mac）**

1. 确认屏幕录制权限已授予
2. 重新构建并运行最新版 JAR

**Q: 画面卡顿？**

1. 画质切为 **「流畅」**
2. 确认内网带宽充足
3. 被控端避免同时运行高负载程序

**Q: 点击后有反应，但画面很久才更新？**

1. Mac 检查屏幕录制权限
2. 重新连接触发全帧刷新
3. 画质切为「流畅」

---

### 键鼠类

**Q: 能看画面但鼠标/键盘控制无效？（Mac）**

检查 **辅助功能** 权限，勾选正确程序后重启被控端。

**Q: 鼠标位置有偏移？**

1. 确认使用最新版本
2. Mac Retina / 多屏场景可能有 1~2 像素偏差
3. 建议单屏测试

**Q: Mac 上 Command 键行为不对？**

工具会自动做 Command ↔ Control 的简单映射。跨系统（Mac 控 Windows 或反向）部分快捷键可能不完全一致。

---

### 运行类

**Q: 双击 `.command` 没反应？**

```bash
chmod +x scripts/*.command
```

或改用终端：

```bash
java -jar target/her-desk-1.0.0.jar server
```

**Q: 提示找不到 JAR？**

先执行 `mvn clean package` 生成 `target/her-desk-1.0.0.jar`。

**Q: 找不到 java 命令？**

安装 JDK 并确认：

```bash
java -version
```
